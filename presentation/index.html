Image and container

Image:

A snapshot of a disk image. Can be created by commiting a container.

Container

A program running in docker, which uses an image as a base image. Each container start from scratch from the base image.

Image layers

Start a container

docker run ubuntu echo "Hello World"

docker run --interactive --tty ubuntu /bin/bash

Restart a container

Will re-run the same command. Will re-use same disk.

docker start --attach --interactive a9f2bd405f9e

root@a9f2bd405f9e:/# echo "Hello World" > /anotherfile.txt
root@a9f2bd405f9e:/# exit

Dockerfiles pt. 2

ADD, ENTRYPOINT, USER AND EXPOSE

ADD

Copy a file from the host to the image


FROM ubuntu-jdk-7

ADD javapp.jar /var/apps/javapp/javapp.jar

ENTRYPOINT

Command to run when starting the container


FROM ubuntu-jdk-7
ADD javapp.jar /var/apps/javapp/javapp.jar

ENTRYPOINT ["java", "-jar", "/var/apps/javapp/javapp.jar"]

Same as running docker insert

USER

Run command as user


FROM ubuntu-jdk-7
ADD javapp.jar /var/apps/javapp/javapp.jar
ENTRYPOINT ["java", "-jar", "/var/apps/javapp/javapp.jar"]

USER daemon

EXPOSE

Specify which ports the container should make available


FROM ubuntu-jdk-7
ADD javapp.jar /var/apps/javapp/javapp.jar
ENTRYPOINT ["java", "-jar", "/var/apps/javapp/javapp.jar"]
USER daemon

EXPOSE 4567

Publish ports

Ports are not automatically published

docker run -p 8888:4567 ubuntu-myapp

Maps ports 8888 on host machine to 4567 on container

Linking containers

Container #1: Redis

FROM ubuntu:latest

RUN apt-get update
RUN apt-get -y install redis-server

ENTRYPOINT ["redis-server"]

EXPOSE 6379
$ docker build -t redis /path/to/Dockerfile
$ docker run -d --name="redis" redis

Redis is exposing 6379 to other containers, but not to the host


$ docker ps
